❌ Lỗi 2: Chỉ phân tích 8000 ký tự đầu tiên của bài báo
Trong processor.py, summarize_text() cắt toàn bộ nội dung PDF còn text[:8000]. Như vậy bạn chỉ đọc được phần tóm tắt/mở đầu, bỏ lỡ phương pháp và kết quả chính.

Fix: Chia nhỏ văn bản thành chunk (vd 2000 token), tóm tắt từng chunk (map-reduce) hoặc dùng LLM có context lớn hơn.

python
def chunk_text(text, max_chars=6000, overlap=200):
    chunks = []
    start = 0
    while start < len(text):
        end = min(start + max_chars, len(text))
        chunks.append(text[start:end])
        start += max_chars - overlap
    return chunks

def summarize_text(self, text, model="llama3"):
    summaries = []
    for chunk in self.chunk_text(text, max_chars=6000):
        prompt = f"Tóm tắt nội dung sau của bài báo khoa học:\n{chunk}"
        response = self._call_llm(prompt, model)
        summaries.append(response)
    # Gọi LLM 1 lần nữa để tổng hợp các bản tóm tắt con
    final = self._call_llm(f"Tổng hợp các ý sau thành một bản tóm tắt duy nhất:\n{' '.join(summaries)}", model)
    return final
❌ Lỗi 3: Hàm trích xuất văn bản PDF dễ thất bại với file scan / tiếng Việt
extract_text_from_pdf dùng pdfminer cơ bản và decode với latin-1, dễ lỗi font Unicode, file scan (ảnh) thì không đọc được.

Fix: Thêm fallback OCR với pytesseract nếu văn bản trích xuất quá ngắn (<100 ký tự). Đồng thời dùng chardet để dò encoding thay vì mặc định latin-1.

python
# cài thêm: pip install pytesseract pdf2image chardet
import chardet
import pytesseract
from pdf2image import convert_from_path

def extract_text_from_pdf(pdf_path):
    # ... code pdfminer cũ ...
    text = ...
    if len(text.strip()) < 100:  # có thể là file scan
        images = convert_from_path(pdf_path, first_page=1, last_page=3)  # vài trang đầu
        text = ''
        for img in images:
            text += pytesseract.image_to_string(img, lang='eng+vie')
    # dò encoding nếu là bytes
    if isinstance(text, bytes):
        encoding = chardet.detect(text)['encoding'] or 'utf-8'
        text = text.decode(encoding, errors='ignore')
    return text

🚀 2. Nâng cấp đề xuất (Upgrades)
a. Sử dụng model mạnh hơn và tối ưu prompt
Thay llama3 bằng llama3.1:8b (hỗ trợ 128K context) hoặc mistral:7b có khả năng phân tích tốt hơn.

Viết prompt có cấu trúc: yêu cầu LLM trả về JSON để dễ xử lý.

Ví dụ prompt cho tóm tắt bài báo:

text
Bạn là trợ lý nghiên cứu. Hãy phân tích bài báo và trả về JSON với các trường:
{
  "title": "...",
  "problem": "...",
  "method": "...",
  "contributions": "...",
  "limitations": "...",
  "future_ideas": "..."
}
Nội dung bài báo: {chunk}
b. Vector hóa cả phần tóm tắt để truy vấn ngữ nghĩa chính xác hơn
Hiện tại database.py lưu cả full_text và summary nhưng chỉ embedding full_text. Nên thêm bộ sưu tập riêng cho summary hoặc lưu summary vào metadata, để truy xuất nhanh khi sinh báo cáo.

d. Chạy thread an toàn và graceful shutdown
main.py dùng schedule.run_pending() trong vòng lặp while True không có cơ chế thoát. Bạn nên bọc trong try/except cho KeyboardInterrupt và đặt thread là daemon.

python
import threading, signal, sys

def run_scheduler():
    while True:
        schedule.run_pending()
        time.sleep(1)

if __name__ == "__main__":
    t = threading.Thread(target=run_scheduler, daemon=True)
    t.start()
    signal.signal(signal.SIGINT, lambda sig, frame: sys.exit(0))
    t.join()
